# PRD: urushi-chronicle

**作成者**: Ryusei | **日付**: 2026-03-29 | **ステータス**: ドラフト
**アイデアID**: 484 | **ドメイン**: 蒔絵/螺鈿 | **難易度**: Medium

## 問題定義

蒔絵・螺鈿は、漆の塗り重ね（下塗り・中塗り・上塗り）の各工程で温度・湿度・乾燥時間が仕上がりに直結する。しかし現在、制作工程の記録は紙のノートや個人の記憶に依存しており、再現性がない。特に漆は「湿度が高いほど早く固まる」という逆説的な性質を持ち、最適な乾燥条件の管理が職人の経験と勘に委ねられている。

既存の制作記録アプリは汎用的で、漆工芸特有の「環境条件と仕上がりの因果関係」を記録・分析する機能がない。蒔絵特化のデジタルツールは市場に存在しない。

## 目標と成功指標

| 目標 | 指標 | ターゲット |
|------|------|----------|
| 制作工程の完全なデジタル記録 | 1作品あたりの記録完了率 | 90%以上の工程が記録される |
| 漆風呂の環境条件自動記録 | IoTセンサーデータ取得率 | 99%以上（5分間隔） |
| 経年変化の長期追跡 | 定点撮影による記録 | 月1回以上の記録継続率80% |
| 最適乾燥条件の可視化 | 過去データからの条件提案精度 | 職人の判断と80%一致 |

## スコープ外

- IoTセンサーハードウェアの製造・販売（既製品のESP32+DHT22を前提）
- AI/MLによる自動品質判定（v2以降）
- 複数工房間のデータ共有・SNS機能（v2以降）
- モバイルネイティブアプリ（PWAで対応）
- 漆の材料調達・在庫管理機能

## ユーザーストーリー

- **蒔絵職人**として、制作中の各工程（下塗り→中塗り→上塗り→蒔絵→螺鈿）を写真・素材・環境条件とともにタイムラインで記録したい。なぜなら次回の制作で同じ条件を再現したいから。
- **蒔絵職人**として、漆風呂の温度・湿度をリアルタイムで確認し、異常値のときにアラートを受け取りたい。なぜなら漆の固化不良を防ぎたいから。
- **蒔絵職人**として、完成作品の経年変化を定点撮影で記録し、漆の艶の変化を時系列で確認したい。なぜなら材料・技法と経年変化の関係を学びたいから。
- **蒔絵教室の指導者**として、生徒の制作記録を閲覧し、各工程へのフィードバックを残したい。なぜなら対面指導の補完として使いたいから。

## 技術要件

### 技術スタック

- **Go**: IoTセンサーデータ収集サービス（MQTT broker連携、goroutineによる高並列処理）
- **TypeScript/React**: タイムライン型制作記録UI（SPA）
- **PostgreSQL + TimescaleDB**: 時系列環境データの格納・クエリ
- **GCP Cloud Run**: API/フロントエンドのホスティング

### アーキテクチャ

```
[ESP32 + DHT22] --MQTT--> [Go IoT Gateway] --REST--> [Go API]
                                                        |
[React SPA] <------------- REST API --------------------|
                                                        |
                                               [PostgreSQL + TimescaleDB]
```

- Go IoT Gateway: MQTTブローカーからセンサーデータを受信し、TimescaleDBに格納
- Go API: 制作工程CRUD、作品管理、環境データクエリを提供するREST API
- React SPA: タイムライン表示、リアルタイムモニタリング、ギャラリー

> **MVP: インメモリストア使用** -- MVPフェーズでは全リポジトリ（環境データ、作品、工程）がインメモリ実装（リングバッファ、100,000エントリ上限）です。PostgreSQL + TimescaleDB への移行は本番デプロイ時に実施予定であり、MVPスコープ外です。

### 非機能要件

- パフォーマンス: API応答 95パーセンタイル < 200ms
- セキュリティ: 認証はJWT、画像アップロードはCloud Storage署名付きURL
- スケーラビリティ: 同時接続10ユーザー（個人〜小規模工房想定）
- 可用性: 99%（Cloud Run SLA準拠）

## 競合分析

| 競合 | URL | ギャップ |
|------|-----|---------|
| なし（汎用制作記録アプリのみ） | - | 漆工芸特有の環境条件記録・漆風呂管理機能が皆無 |
| Notion / Google Sheets | - | IoTセンサー連携なし、時系列データ分析不可 |
| Grafana + InfluxDB | - | 環境モニタリングは可能だが制作工程との紐付けなし |

**差別化ポイント**: 漆風呂の温湿度IoT管理+経年変化トラッキングは蒔絵特化で完全に空白市場。制作工程と環境データを紐付けて記録・分析できる唯一のツール。

## 技術リスク

| リスク | 発生可能性 | 影響 | 軽減策 |
|--------|----------|------|--------|
| IoTハードウェア連携の信頼性 | 中 | 高 | ESP32+DHT22は実績あり。MVPではモックデータでも動作する設計にする |
| TimescaleDB運用の複雑さ | 低 | 中 | Cloud SQL for PostgreSQL + TimescaleDB拡張。マネージド利用 |
| 画像ストレージコスト | 低 | 中 | Cloud Storage + CDN。画像圧縮をフロントで実施 |
| MQTTブローカーの運用 | 中 | 中 | MVPではCloud IoT Core代替としてEMQXのマネージド版を検討 |

## マイルストーン

### Phase 0: MVP（2週間）

1. **制作工程タイムライン記録** — 工程名・写真・素材・環境条件のCRUD
2. **漆風呂の温湿度リアルタイムモニタリングとアラート** — IoTデータ受信・表示・閾値アラート
3. **完成作品ギャラリーと経年変化の定点撮影記録** — 作品一覧・時系列比較表示

### Phase 1: 拡張（1ヶ月）

- 過去データからの最適乾燥条件提案
- 複数漆風呂の管理
- データエクスポート（CSV/PDF）

### Phase 2: 成長（3ヶ月）

- 工房間のデータ共有
- モバイルPWA最適化
- AI画像分析による艶評価

## マネタイズ
